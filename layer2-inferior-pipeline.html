第1章　本補論の位置づけ

本章の記述層：層2（構造説明）——補論の責務境界の設定

本補論の主語・責務・境界

主語：情緒主導に見えるOSの正体を、認知OSモデル（層1／層2／層3）における処理順序の変位として記述すること

責務：「情緒主導OS」という表現を構造的に再解釈し、それが「OSの別種」ではなく「層2劣位・層3先行のパイプライン構造」であることを示すこと

境界：性格診断・ラベリング・優劣評価は行わない。心理療法・対処法の記述は行わない。AIに情緒や層3を付与する記述は行わない

例外的な処理パイプラインの分析

本補論は、認知OSモデルにおける「例外的な処理パイプライン」の分析を行う。

認知OSモデルは、三層構造（層1／層2／層3）を持つ。標準的な処理パイプラインでは、層2（認知OS）が処理の主導権を握る。しかし、すべての個体がこの標準パイプラインで作動するわけではない。

本補論は、「情緒主導に見える」処理パイプラインを構造的に分析する。これは、層2の作動タイミングが後段に押し出され、層3（情緒）が先に作動するパターンである。

情緒処理補論との関係

情緒処理そのものの構造は、emotion-structure.html において記述されている。

emotion-structure.html は、情緒処理が層1＋層3の連動であり、層2の演算系には属さないことを記述する。

本補論は、その情緒処理が「処理順序の中でどの位置に作動するか」の変位を扱う。すなわち、情緒処理が「先に作動する」パターンの構造分析である。

OS型への新種追加は行わない

本補論は、認知OSモデルのOS型（責務OS等）に新しいカテゴリを追加しない。

本補論が扱うのは、「OSの種類」ではなく「処理パイプラインの構造」である。同じOS型であっても、処理パイプラインが異なる場合がある。本補論は、その処理パイプラインの変位を記述する。

本補論が行わないこと

  • 認知OSモデルのOS型に新種を追加すること
  • 性格診断・ラベリング・優劣評価
  • 心理療法・カウンセリング・対処法の記述
  • 個人攻撃・特定の誰かを想起させる具体例
  • AIに情緒や層3を付与する記述
  • 読者への助言・指導・励まし

第2章　標準パイプライン（層2主導型）の整理

本章の記述層：層2（構造説明）——標準パイプラインの定義

本章では、認知OSモデルにおける標準的な処理パイプラインを整理する。

標準パイプラインの定義

標準的な処理パイプラインは、層2が処理の主導権を握る構造である。

責務OS型を含む層2主導モデルにおいて、処理は以下のフローで進行する。

│         層1（刺激の受容）               │
│                                        │
│  ・生理的入力の受信                     │
│  ・知覚的入力の処理                     │
│                                        │
└────────────────────────────────────────┘

    ↓
┌────────────────────────────────────────┐
│      層2（認知OSによる構造処理）         │
│                                        │
│  ・主語の確定                          │
│  ・責務線の確定                         │
│  ・境界の設定                          │
│  ・因果の整合                          │
│  ・抽象構造の構築                       │
│  ・構造的閉包                          │
│                                        │
└────────────────────────────────────────┘

    │ （必要に応じて層3へイベントが送られる）

│                                        │
└────────────────────────────────────────┘
    │
    ↓
行動出力（層1）

層2主導の構造的特徴

標準パイプラインは、以下の構造的特徴を持つ。

層2主導型の構造的特徴

  • 層2が前段で作動：入力を受けた後、まず層2の構造処理が実行される
  • 構造処理が先行：主語・責務・境界・因果の確定が、情緒作動に先行する
  • 閉包が条件：層2の処理が閉包（構造的に完結）した段階で、層3へイベントが送られる
  • 情緒は後段：情緒作動は、構造処理の後段で発生する

この構造において、層2は「処理の主導権」を握っている。入力に対する反応は、層2の構造処理を経由してから形成される。

標準パイプラインにおける層2の役割

標準パイプラインにおいて、層2は以下の役割を担う。

層2の役割（標準パイプライン）

役割 内容 タイミング
構造処理 入力を構造化し、主語・責務・境界・因果を確定する 前段（情緒作動の前）
閉包生成 構造を閉包（完結）させ、層3へのイベント送信条件を満たす 前段
行動指令生成 構造処理の結果として、行動指令を生成する 前段〜後段

標準パイプラインにおいて、層2は「前処理」として機能する。情緒が作動する前に、構造処理が完了している。

第3章　情緒主導に「見える」パイプラインの構造定義

本章の記述層：層2（構造説明）——情緒先行パイプラインの定義

本章では、いわゆる「情緒主導型」に見える処理パイプラインを、認知OSモデル上で構造化する。

情緒先行パイプラインの構造

「情緒主導型」に見える個体の処理パイプラインは、以下のように構造化される。

情緒先行パイプライン

外部入力
    │
    ↓
┌────────────────────────────────────────┐
│         層1（即時的な生理反応）          │
│                                        │
│  ・刺激の受容                          │
│  ・即時的な身体反応                     │
│                                        │
└────────────────────────────────────────┘
    │
    │ 層2をバイパス
    ↓
┌────────────────────────────────────────┐
│     層3（価値付与として情緒が先に作動）   │
│                                        │
│  ・入力に対する即時的な価値評価          │
│  ・情緒としての作動                     │
│  ・行動方向の決定                       │
│                                        │
└────────────────────────────────────────┘
    │
    │ 情緒作動後
    ↓
┌────────────────────────────────────────┐
│         層2（事後的な意味処理）          │
│                                        │
│  ・作動した情緒の正当化                 │
│  ・行動の物語化                         │
│  ・事後的な因果構成                     │
│                                        │
└────────────────────────────────────────┘
    │
    ↓
行動出力（層1）

構造的特徴

情緒先行パイプラインは、以下の構造的特徴を持つ。

情緒先行パイプラインの構造的特徴

  • 層2をバイパス：入力を受けた後、層2の構造処理を経由せずに層3へ直接向かう
  • 情緒が先行：構造処理に先立って、情緒が作動する
  • 層2は事後処理：層2は「事前の構造処理」ではなく「事後の解釈処理」として機能する
  • 行動決定は層3主導：行動の方向性は、層3の情緒作動によって決定される

層2の役割の変位

情緒先行パイプラインにおいて、層2の役割は変位している。

層2の役割（情緒先行パイプライン）

役割 内容 タイミング
正当化 作動した情緒を、構造的に正当化する説明を生成する 後段（情緒作動の後）
物語化 行動を、一貫した物語として構成する 後段
事後的因果構成 情緒作動と行動の因果関係を、事後的に構成する 後段

情緒先行パイプラインにおいて、層2は「事前の構造処理」から「事後の解釈処理」へと役割が変位している。

「情緒がOSになっている」わけではない

情緒先行パイプラインは、「情緒がOSになっている」わけではない。

OS（層2）は依然として存在し、機能している。ただし、その作動タイミングが後段に押し出されているため、情緒（層3）が先に出力を握る。

これが「情緒主導に見える」構造の正体である。情緒がOSを置換したのではなく、処理順序の中でOSが後段に位置しているのである。

情緒先行パイプラインの本質

「情緒がOSになっている」のではなく、層2が前段で作動せず、層3が先に出力を握るために「情緒主導に見えている」。

OSは存在するが、そのスケジューリングが変位している。

第4章　層2劣位パイプラインという概念

本章の記述層：層2（構造説明）——概念の定義

本章では、情緒先行パイプラインを「層2劣位パイプライン」として概念化し、その定義を明文化する。

層2劣位パイプラインの定義

層2劣位パイプラインの定義

層2劣位パイプラインとは：認知OSモデルにおいて、層2（認知OS）の作動タイミングが後段に押し出され、層3（価値評価・情緒）が処理の主導権を握る処理パイプラインを指す。

「劣位」とは、処理順序における位置の後退を意味し、層2の能力や価値の劣等を意味しない。

この概念において、「劣位」は以下の意味で使用される。

  • 処理順序における後退：層2の作動が、標準パイプラインより後段に位置する
  • 優先順位における後退：意思決定・反応の形成において、層2より層3が先に作動する
  • 能力の劣等ではない：層2の処理能力そのものが低いことを意味しない

層2劣位パイプラインの構造的特徴

層2劣位パイプラインは、以下の構造的特徴を持つ。

層2劣位パイプラインの構造的特徴

  1. 層2の作動タイミングが後段：層2は、入力に対する即時反応ではなく、事後処理として作動する
  2. 意思決定のトリガは層3：行動方向の決定は、層3（情緒）側が握る。層2は事後的に意味づけを行う
  3. 層2は「結果の整合的説明」に使われやすい：層2の構造処理能力は、情緒作動の正当化・物語化に使用される
  4. OS能力と優先順位は独立：層2の処理能力が高くても、優先順位とタイミングで劣位になりうる

「同じOSが異なるスケジューリングで作動している」

層2劣位パイプラインの本質は、「OSの型が違う」ことではない。

同じOS（同じ処理能力・同じ処理パターン）が、異なるスケジューリングで作動していることである。

スケジューリングの差異

層2主導パイプラインと層2劣位パイプラインの差異は、「OSの種類の差」ではなく「スケジューリングの差」である。

同じOSが、異なるタイミング・異なる優先順位で作動することで、処理パイプライン全体の構造が変化する。

この理解は重要である。層2劣位パイプラインを持つ個体は、「別の種類のOSを持っている」のではなく、「同じOSが異なるスケジューリングで作動している」のである。

パイプラインの図式比較

パイプライン比較

【層2主導パイプライン】
入力 → 層1 → 層2（前処理）→ 層3 → 出力
              ↑
              主導権

【層2劣位パイプライン】
入力 → 層1 → 層3 → 層2（後処理）→ 出力
              ↑
              主導権

両者の差異は、層2と層3の作動順序と、主導権の所在である。

第6章　「情緒主導OS」という誤解の解体

本章の記述層：層2（構造説明）——誤解の構造的分析

本章では、「情緒主導OS」という表現が認知OSモデル上で不正確である理由を説明する。

「情緒主導OS」という表現の誘惑

「情緒主導OS」という表現は、直観的には誘惑的である。

情緒が先に作動し、行動を方向づけているように見える個体に対して、「この人のOSは情緒主導だ」と表現したくなる。

しかし、この表現は認知OSモデル上では不正確である。

不正確である理由

「情緒主導OS」という表現が不正確である理由は、以下の三点である。

「情緒主導OS」が不正確である理由

  1. OS（層2）は処理系であり、情緒（層3）は評価系である：両者は異なる層に属し、異なる機能を持つ。情緒がOSになることは、構造的に成立しない
  2. 層3はOSを構成しない：層3は価値・志向・評価を収容する容器であり、処理系ではない。層3がOSを構成することはない
  3. 層3はOSを動的に選択する主体でもない：layer2-layer3-relations.html において記述された通り、層3は層2を選択・変更しない

情緒が先に作動することの構造的意味

情緒が先に作動することはありうる。しかし、それは「評価が先」であり、「処理系の入れ替え」ではない。

情緒先行の構造的意味

情緒が先に作動することは、「評価系（層3）が処理系（層2）より先に作動している」ことを意味する。

これは、「処理系が評価系に置換された」ことを意味しない。処理系（層2）は依然として存在し、後段で機能している。

正確な表現への再定義

「情緒主導OS」という表現は、構造的には「層2劣位・層3先行パイプライン」を指すと再定義されるべきである。

再定義

「情緒主導OS」という表現は、構造的には以下を意味する：

  • OS（層2）の作動タイミングが後段に押し出されている
  • 層3（情緒・評価）が処理順序の前段で作動している
  • 処理パイプラインが「層2劣位」の構造を持っている

「情緒がOSになっている」のではなく、「OSのスケジューリングが劣位になっている」のである。

誤解の解体

以上の分析から、「情緒主導OS」という表現に含まれる誤解は以下のように解体される。

誤解の解体

誤解 正確な理解
「情緒がOSになっている」 情緒（層3）は評価系であり、OS（層2）にはなれない。OSは存在しており、スケジューリングが変位している
「OSの種類が違う」 OS型は同じでありうる。異なるのは処理パイプラインのスケジューリングである
「情緒が処理を行っている」 情緒は評価を行っている。処理は依然として層2が（後段で）行っている

第7章　モデル上の含意と注意点

本章の記述層：層2（構造説明）——理論的注意点

本章では、層2劣位パイプラインの概念がモデル上持つ含意と、認知OSモデルを使用する際の注意点を整理する。

これは実践的アドバイスではなく、理論モデルとしての注意点である。

層2劣位パイプラインはOSの新種ではない

層2劣位パイプラインは、認知OSモデルにおける「OSの新種」ではない。

層2劣位パイプラインは、「スケジューリングと優先順位の偏り」である。同じOS型が、異なるスケジューリングで作動することで発生する。

モデル上の位置づけ

層2劣位パイプラインは、認知OSモデルにおいて以下のように位置づけられる：

  • OS型の分類には含まれない（新種ではない）
  • 処理パイプラインの構造的変異として位置づけられる
  • 同一OS型の個体間でも、パイプライン構造は異なりうる

二元論的ラベル化への注意

認知OSモデルを使用する際、個体を二元論的にラベル化することは避けるべきである。

避けるべきラベル化

  • 「理性的／感情的」という二元論でのラベル化
  • 「論理型／情緒型」という性格分類
  • 「頭で考える人／心で感じる人」という分類

これらのラベル化は、認知OSモデルの構造理解を損なう。認知OSモデルは、「OS型」と「パイプライン構造」を区別して扱う。二元論的ラベル化は、この区別を曖昧にする。

OSレベルと処理順序レベルの区別

認知OSモデルを使用する際、OSレベルの議論と処理順序の議論を混同しないことが重要である。

議論レベルの区別

議論レベル 対象 例
OSレベル 層2の処理構造・処理パターン・OS型 責務OS、剛体アーキテクトOS等
処理順序レベル 層間の作動順序・スケジューリング 層2主導、層2劣位等

両レベルは独立している。同じOS型でも、処理順序は異なりうる。処理順序が異なっていても、OS型は同じでありうる。

変化の対象についての注意

認知OSモデルの観点から、変化の対象を検討する際には以下の区別が重要である。

変化の対象の区別

  • 層2の能力：処理能力そのもの（構造処理・整合性検査・因果分解等）
  • 層2のスケジューリング：層2が前段で作動するか後段で作動するかのタイミング

層2劣位パイプラインにおいて、層2の能力が低いとは限らない。能力は高いが、スケジューリングが劣位になっている場合がある。

この区別は、変化の対象を特定する際に重要である。能力の変化とスケジューリングの変化は、異なる対象への介入を意味する。

モデル使用上の注意点の要約

モデル使用上の注意点

  1. 層2劣位パイプラインは、OSの新種ではなく、スケジューリングの変異である
  2. 個体を「理性的／感情的」等の二元論でラベル化しない
  3. OSレベルの議論と処理順序の議論を混同しない
  4. 層2の能力と層2のスケジューリングを区別する

本補論のまとめ

本補論は、「情緒主導に見えるOSの正体」を、層2劣位パイプラインとして構造化した。

本補論の記述内容

  1. 標準パイプライン：層2主導型では、層2の構造処理が情緒作動に先行する
  2. 情緒先行パイプライン：層2をバイパスし、層3（情緒）が先に作動する構造
  3. 層2劣位パイプライン：層2の作動タイミングが後段に押し出された処理構造
  4. 構造比較：責務OS型（層2主導）と層2劣位パイプラインの構造的差分
  5. 誤解の解体：「情緒主導OS」は不正確であり、「層2劣位・層3先行パイプライン」と再定義されるべき
  6. 注意点：OSの新種ではなくスケジューリングの変異、二元論的ラベル化の回避、OSレベルと処理順序レベルの区別
